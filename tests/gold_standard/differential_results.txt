Note on mips64: the mips64 backend emits big-endian MIPS64 ELF, but the only
  available QEMU binary is qemu-mips64el (little-endian). QEMU will refuse
  the resulting binaries with an 'Invalid ELF' error. MIPS numbers in this
  report are therefore not real execution results — they reflect QEMU's ELF
  rejection (typically exit code 1). See Task 9-b in worklog.md for context.

  involving function calls frequently crash or time out on this backend.
Note on ppc64: documented Address-typed-return calling-convention bug
  (Task 9-b); *_func_load / struct2_func_load / *_address_return tests
  typically exit 0 on ppc64.
