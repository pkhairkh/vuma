
================================================================================
  LOONGARCH64 COMPLETE INSTRUCTION OPCODE ENCODING TABLE
================================================================================
  
  Sources:
  - Linux kernel: arch/loongarch/include/asm/inst.h (torvalds/linux master)
  - Community: github.com/loongson-community/loongarch-opcodes (develop branch)
  - LoongArch Reference Manual Vol1 (loongson.github.io)
  
  FORMAT: The "Opcode" column is the value to place in the opcode field of
  the instruction encoding. The "Full enc" column shows the complete 32-bit
  instruction word with all register/immediate fields set to zero.
  
  BIT LAYOUT per format (LSB-first):
    reg0i15:  [14:0]=imm15    [31:15]=opcode17
    reg0i26:  [15:0]=imm_lo   [25:16]=imm_hi  [31:26]=opcode6
    reg1i20:  [4:0]=rd        [24:5]=imm20    [31:25]=opcode7
    reg1i21:  [4:0]=imm_lo5   [9:5]=rj        [25:10]=imm_hi16 [31:26]=opcode6
    reg2:     [4:0]=rd        [9:5]=rj        [31:10]=opcode22
    reg2i5:   [4:0]=rd        [9:5]=rj        [14:10]=imm5     [31:15]=opcode17
    reg2i6:   [4:0]=rd        [9:5]=rj        [15:10]=imm6     [31:16]=opcode16
    reg2i12:  [4:0]=rd        [9:5]=rj        [21:10]=imm12    [31:22]=opcode10
    reg2i14:  [4:0]=rd        [9:5]=rj        [23:10]=imm14    [31:24]=opcode8
    reg2i16:  [4:0]=rd        [9:5]=rj        [25:10]=imm16    [31:26]=opcode6
    reg2bstr: [4:0]=rd        [9:5]=rj        [15:10]=lsb      [21:16]=msb [31:22]=opcode10
    reg3:     [4:0]=rd        [9:5]=rj        [14:10]=rk       [31:15]=opcode17
    reg3sa2:  [4:0]=rd        [9:5]=rj        [14:10]=rk       [16:15]=sa2  [31:17]=opcode15

  IMPORTANT NOTES FOR COMPILER BACKEND IMPLEMENTORS:
  
  1. ENCODING METHOD: To encode an instruction, use:
     instruction_word = (opcode << opcode_shift) | (fields...)
     where opcode_shift is the format's opcode bit position.
     
  2. The FCMP instructions encode the condition code as part of the 17-bit
     opcode field. Each condition variant has a unique opcode value.
     The 12-bit "top opcode" (bits 31:20) is:
       FCMP.S: 0x0C1 (conditions increment by 1 in bits 9:5)
       FCMP.D: 0x0C2 (conditions increment by 1 in bits 9:5)
     
  3. The FMA (fused multiply-add) 4R format has:
     12-bit "top opcode" (bits 31:20):
       FMADD.S:  0x081   FMADD.D:  0x082
       FMSUB.S:  0x085   FMSUB.D:  0x086
       FNMADD.S: 0x089   FNMADD.D: 0x08A
       FNMSUB.S: 0x08D   FNMSUB.D: 0x08E
  
  4. The FSEL instruction 4R format:
     12-bit "top opcode" (bits 31:20): 0x0D0
  
  5. Branch instructions (beq, bne, blt, bge, bltu, bgeu):
     NOTE: The community opcode list uses "bgt/ble/bgtu/bleu" as the 
     primary mnemonic with "orig_name=blt/bge/bltu/bgeu", because the
     encoding semantics follow MIPS convention (the branch tests rj < rd).
     The kernel uses the LoongArch manual convention (blt/bge).
     The OPCODE VALUES are the same regardless of naming convention.
  
  6. The "Full enc" values assume all register and immediate fields are zero.
     To construct an actual instruction:
     - For reg3: (opcode << 15) | (rk << 10) | (rj << 5) | rd
     - For reg2i12: (opcode << 22) | (imm12 << 10) | (rj << 5) | rd
     - For reg2i16: (opcode << 26) | (imm16 << 10) | (rj << 5) | rd
     - etc.
