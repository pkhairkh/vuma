   296→### Pipeline Changes
   297→The compilation pipeline is now:
   298→```
   299→Source → Parse → AST → SCG → BD Inference → MSG → IVE Verification
   300→       → SCG Transforms → IR Lowering → RegAlloc → Code Emission → COR Init
   301→```
   302→
   303→**CorInit stage** does the following:
   304→1. Bridges the `vuma_scg::SCG` to COR's internal representation using `CORuntime::from_vuma_scg()`
   305→2. Creates a `Delta` containing all node IDs from the compiled SCG
   306→3. Calls `compile_incremental()` on the CORuntime to establish the always-compiled invariant
   307→4. Stores the `CORuntime` in `CompilationOutput.cor_runtime`

   425→### Root Causes and Fixes
   426→
   427→#### Bug 1: Resource ID Mismatch (liveness extraction)
   428→- **Problem**: `extract_liveness_input` assigned `ResourceId(next_resource_id)` (counter-based) to allocation events but `ResourceId(dealloc.allocation_node.as_u64())` (node-ID-based) to deallocation events. These never matched, so the liveness checker couldn't pair allocations with deallocations.
   429→- **Fix**: Added `alloc_node_to_rid` HashMap to map allocation NodeIds to their assigned ResourceIds. Deallocation events now look up the correct ResourceId from this map.
   430→
   431→#### Bug 2: Missing CFG Edges (liveness extraction)
   432→- **Problem**: The original code iterated over SCG edges but never added them to the liveness input's CFG, so the liveness checker had no path information and always reported leaks.
   433→- **Fix**: Added ControlFlow edges to the liveness CFG. Initially added ALL edges, but this caused false positives (see Bug 3). Final version only adds ControlFlow edges that don't involve Control nodes.
   434→
   435→#### Bug 3: Derivation/DataFlow Edges Creating False Paths
   436→- **Problem**: Including Derivation and DataFlow edges in the liveness CFG and cleanup graph created "shortcut" paths that bypassed intermediate operations. For example, in `alloc_a → alloc_b → dealloc_a → dealloc_b` with a Derivation edge `alloc_b → dealloc_b`, the path `alloc_a → alloc_b → dealloc_b` bypassed `dealloc_a`, causing a false ConditionalDeallocation report.
   437→- **Fix**: Only ControlFlow edges are now added to the liveness CFG and cleanup graph, since only they represent actual execution ordering.
   438→
   439→#### Bug 4: Control Nodes Creating Dangling Branches
   440→- **Problem**: FunctionEntry/FunctionReturn Control nodes create branches in the CFG that don't connect back to the main control flow (the SCG doesn't model call-return edges). These dangling branches cause the liveness checker and cleanup verifier to report false leaks on paths ending at FunctionReturn nodes.
   441→- **Fix**: ControlFlow edges involving Control nodes (FunctionEntry/Return, Branch, Join, etc.) are now excluded from both the liveness CFG and the cleanup graph.
   442→
   443→#### Bug 5: Overly Aggressive Path-Sensitive Leak Detection
   444→- **Problem**: The liveness checker's `check_resource_leaks` flagged any reachable point that didn't reach a deallocation, even if that point had successors leading to a deallocation. This was too conservative.
   445→- **Fix**: Changed the path-sensitive analysis to only flag dead-end nodes (no successors in the CFG) as potential leak endpoints. Nodes with successors are safe because they transitively reach the deallocation.
   446→
   447→#### Test Assertion Updates
   448→- **framework.rs `test_verify_program_returns_five_invariants`**: Changed from asserting `OverallVerdict::Inconclusive` to asserting `!= OverallVerdict::Fail`, since well-formed programs now get real results (Pass/Proven) instead of placeholder Inconclusive.
   449→- **full_pipeline.rs `test_full_pipeline_read_write_region`**: Changed from checking for Access nodes with Read/Write modes to checking for Computation nodes, since the parser treats `write()` and `read()` as generic function calls rather than typed access operations.

  1444→1. Check for empty input → EmptyInput error
  1445→2. Check for conflicting conclusions (Proven + Refuted) → ConflictingConclusions error

  1447→4. Remap each proof's fact ids sequentially via `remap_proof_ids`
  1448→5. Collect all remapped steps into compound proof

  1455→4. Preserve original conclusion

  1522→### Affected Invariant Computation Rules
  1523→| Invariant | Trigger |
  1524→|-----------|---------|
  1525→| Liveness | Regions or accesses changed |
  1526→| Origin | Derivations changed |
  1527→| Bounds | Derivations or regions changed |
  1528→| Exclusivity | Edges added or removed |
  1529→| Cleanup | Nodes removed or regions affected |
  1530→
  1531→### Incremental Verification Algorithm
  1532→1. If no changes → skip all invariants (savings_ratio = 1.0)
  1533→2. Compute affected invariants from ChangeSet
  1534→3. Invalidate cache entries for affected regions
  1535→4. Re-verify affected regions and update cache
  1536→5. Re-verify affected derivations (cascade to accesses)
  1537→6. Process edge changes for exclusivity
  1538→7. Combine with cached results for unaffected regions
  1539→8. Compute savings_ratio = 1.0 - (nodes_re_checked / total_nodes)

  1565→1. **`compute_affected_invariants` is a static method** — Doesn't need `self`, making it easy to call from `incremental_verify` without needing a `ChangeDetector` instance.

  1675→### Inference Rules Implemented
  1676→
  1677→**RepD Inference:**
  1678→- AllocationNode: Byte(size, align) from payload; Ptr when 8-byte+8-align with pointer successor
  1679→- AccessNode: Byte from access_size/offset alignment
  1680→- CastNode: RepD from target type name (via BDInferenceEngine::repd_from_type_name)
  1681→- ComputationNode: RepD from result_type or inherited from predecessor
  1682→- Second-pass refinement: multiple accesses at different offsets on same allocation → Struct
  1683→
  1684→**CapD Inference:**
  1685→- Read-only access → Read only
  1686→- Write/ReadWrite access → Read+Write
  1687→- Data reaching EffectNode → Read+Write+Persist (backward BFS from Effect nodes)
  1688→- Data crossing security boundary → Read+Compare
  1689→- Data in ptr arithmetic → Compute+DerivePtr
  1690→- Deallocation → Drop only
  1691→
  1692→**RelD Inference:**
  1693→- DataFlow edge → AliasDep (source) + DataDep (target) + Containment if target is Access
  1694→- Derivation edge → DataDep
  1695→- Annotation edge → Equivalence
  1696→- ControlFlow edge → ControlDep
  1697→- Same region membership → Containment
  1698→- Node-type-specific: Deallocation→Liveness, Effect→ControlDep, Computation→DataDep
  1699→
  1700→**Consistency Checking:**
  1701→1. SizeMismatch: Allocation RepD size ≠ allocation payload size
  1702→2. CapabilityViolation: Read-only Access has Write, or ReadWrite missing Read/Write, or Deallocation has Read/Write/DerivePtr/Execute
  1703→3. RelationContradiction: RelD with contradictory temporal relations
  1704→4. FlowViolation: RepD size changes across DataFlow without Cast, or CapD gains capabilities
