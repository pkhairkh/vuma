   220→    - Current EL with SP0: sync, irq, fiq, serror
   221→    - Current EL with SPx: sync, irq, fiq, serror
   222→    - Lower EL AArch64: sync, irq, fiq, serror

   225→  - **`_start` entry point** — Naked assembly in `.text.boot` section:
   226→    1. Saves DTB pointer (x0 → x6) before any clobber
   227→    2. Reads core ID from `MPIDR_EL1.Aff0`
   228→    3. Parks secondary cores (1–3) in `WFE` loop
   229→    4. Sets up 16 KiB stack for core 0 above `__bss_end`, 16-byte aligned
   230→    5. Zeros BSS section (8-byte stores from `__bss_start` to `__bss_end`)
   231→    6. Installs exception vector table via `VBAR_EL1`
   232→    7. Restores DTB pointer and jumps to `boot_main`
   233→  - **`kernel_entry`** — `KERNEL_ENTRY = 0x80000`, the Pi 5 bootloader load address
   234→  - **FDT parsing** — `FdtHeader` struct with `from_raw()`, `from_bytes()`, `is_valid()`:
   235→    - Parses all 10 u32 fields from the 40-byte DTB header
   236→    - Correct mapping: words[0]=magic, [1]=totalsize, [2]=off_dt_struct, [3]=off_dt_strings, [5]=version, [8]=size_dt_strings, [9]=size_dt_struct
   237→    - `is_valid()` checks: magic match, positive totalsize, offsets within bounds, struct/strings don't overflow
   238→  - **`BootInfo`** — Parsed boot context: dtb_addr, FdtHeader, boot_core
   239→  - **`boot_main()`** — High-level entry (core 0 only):
   240→    - Saves DTB address to global
   241→    - Initialises UART at 115200 baud
   242→    - Parses FDT header, logs result
   243→    - Constructs `BootInfo` and calls user `main()`
   244→    - Parks core 0 if main() returns
   245→  - **`zero_bss()`** — Standalone BSS zeroing with SeqCst fence
