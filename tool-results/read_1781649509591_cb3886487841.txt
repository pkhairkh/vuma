  1331Ôćĺ
  1332ÔćĺRoot cause: Wave 1-b correctly made `resolve_expr` return `UnknownVariable`
  1333Ôćĺinstead of silently substituting 0. This exposed two real issues in how the
  1334ÔćĺSCGÔćĺIR builder populates its `names` map:
  1335Ôćĺ
  1336Ôćĺ  (a) **Use-before-def ordering.** The SCGÔćĺcodegen bridge
  1337Ôćĺ      (`pipeline::bridge_scg_to_codegen`) walks the control-flow graph and
  1338Ôćĺ      emits a flat statement list per function. DataFlow-only nodes (those
  1339Ôćĺ      with no incoming ControlFlow edge) are appended *after* the main body
  1340Ôćĺ      via the "remaining nodes" cleanup, so a statement that references
  1341Ôćĺ      `v_<id>` via DataFlow can appear in the flat list *before* the
  1342Ôćĺ      statement that defines `v_<id>`. `lower_statements` iterated in raw
  1343Ôćĺ      order, so the use hit `resolve_expr` before the def was registered.
  1344Ôćĺ
  1345Ôćĺ  (b) **Cross-function DataFlow references.** The bridge's "remaining nodes"
  1346Ôćĺ      cleanup creates a separate synthetic `main` function for unconsumed
  1347Ôćĺ      nodes. If a node N is DataFlow-only and referenced by a statement in

  1353Ôćĺ
  1354Ôćĺ  1. **`topological_sort_statements`** ÔÇö extended the dependency-edge
  1355Ôćĺ     computation to handle use-before-def. Previously it only searched for
  1356Ôćĺ     the *last definition before j*; now, if none is found, it also searches

  1358Ôćĺ     edge, so Kahn's algorithm lowers the def before the use. Existing
  1359Ôćĺ     in-order behavior (and the 3 existing topo-sort unit tests) is
